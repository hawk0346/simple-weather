# スマホ・PC対応 天気検索アプリ設計プラン

## 1. 画面構成

### 検索画面（メイン）
- 都市名入力欄＋検索ボタン（大きめ・片手で押しやすい位置）
- 検索履歴リスト（下部またはドロワー式）
- ダークモード切替ボタン（ヘッダー右上など）
- 検索結果表示（天気・気温・アイコン等）

### PWA対応
- ホーム画面追加用の案内
- ※オフライン時は音声・検索ともに非対応（オフライン対応は不要）

## 2. 機能詳細
- 履歴保存：localStorage（またはIndexedDB）で端末ごとに保存
- 履歴からタップで再検索
- 履歴の削除（全削除・個別削除）
- ダークモード：OS連動＋手動切替（状態はlocalStorage保存）
- レスポンシブ：スマホ・PC両対応、スマホは片手操作重視
- ボタン・入力欄はタップしやすいサイズ

## 3. UI/UX改善ポイント
- ボタン・入力欄の高さ48px以上
- 主要操作は画面下寄せ（親指で届きやすい）
- 検索履歴はスワイプやタップで簡単操作
- ダークモード切替はワンタップ
- 余計な装飾は省き、シンプルで見やすい配色

## 4. 技術方針
- React＋Tailwind CSSでUI実装
- PWA対応（オフライン時は機能制限）
- スマホ・PC両方で動作（主要ブラウザ対応）
